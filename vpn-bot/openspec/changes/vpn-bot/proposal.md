## Why

В репозитории vpn-bot уже есть рабочий Go-сервис (Telegram + SQLite + Hiddify), но нет формализованных требований в OpenSpec: неочевидны границы возможностей, сценарии админа и пользователя и критерии приёмки. Сейчас удобно зафиксировать контракт поведения «как есть», чтобы дальнейшие доработки опирались на общий источник правды.

Дополнительно требуется зафиксировать новый пользовательский UX выдачи конфигов: адреса Telegram Proxy через домен `users.*` (вместо "сухого" IP), раздельные VPN-ссылки (WireGuard / Full Xray / все конфиги), self-service команды пользователя для перевыпуска и статистики, и визуальные карточки выдачи.

## What Changes

- Добавить набор capability-спек под текущие сценарии бота (пользовательский запрос доступа, модерация, Hiddify, статус, отзыв).
- Добавить `design.md` с обзором модулей, данных и интеграций (без обязательства менять код в рамках этого change — зависит от `tasks.md`).
- Добавить `tasks.md` с пошаговой проверкой соответствия кода спекам и точечными задачами при расхождениях.
- Зафиксировать правила выдачи ссылок:
  - Telegram Proxy ссылка должна использовать 4-й уровень домена с префиксом `users.`.
  - Вместо одной общей VPN-ссылки отправлять три: WireGuard, Full Xray и список всех конфигов.
  - Для WireGuard и Full Xray добавить кнопки скачивания готового файла конфига.
  - По нажатию бот сам скачивает содержимое по исходной ссылке и отдает пользователю текстовый файл формата `<user>_<protocol>.txt`.
- Добавить пользовательские self-service команды:
  - команда перевыпуска (`/revoke` как canonical, с поддержкой alias `/revok`),
  - команда `stats` с минимальной статистикой трафика.
- Добавить визуальную выдачу ссылок через картинки (фон в стиле `arengate-landing`) с рендером текста через `golang.org/x/image`:
  - карточка "VPN" + список VPN-вариантов,
  - карточка "Telegram Proxy" + proxy-ссылка,
  - финальное сообщение с тегом владельца для обратной связи.

## Capabilities

### New Capabilities

- `user-access-request`: `/start`, кнопка запроса доступа, запись в БД в состоянии ожидания, проверки для `banned` / уже `active` без секрета.
- `admin-moderation`: команды `/stats`, `/users`, `/approve`, callback-кнопки одобрения/отклонения, ограничение по `ADMIN_IDS`.
- `hiddify-provisioning`: создание пользователя в Hiddify, срок пакета и лимит трафика из конфига, сохранение UUID и активация в SQLite.
- `user-subscription-status`: `/status` и выдача ссылок VPN/MTProxy для активного пользователя с UUID.
- `subscription-revocation`: отзыв доступа (админ/сервис) в согласовании с Hiddify и БД.
- `connection-link-delivery`: формирование и выдача ссылок подключения (WireGuard, Full Xray, all configs, Telegram Proxy на домене `users.*`) в корректной последовательности сообщений.
- `rich-media-cards`: генерация визуальных карточек "VPN" и "Telegram Proxy" в Go с использованием `golang.org/x/image` и бренд-фона.
- `user-self-service`: команды пользователя для перевыпуска своих конфигов и просмотра минимальной статистики трафика.

## Impact

- Код: `main.go`, `bot/handlers/*`, `store/*`, `services/hiddify`, `services/approve`, `services/revoke`, `config`, миграции SQLite.
- Внешние системы: Telegram Bot API (long poll или webhook), панель Hiddify по HTTP API.
- Конфигурация: переменные окружения (`BOT_TOKEN`, `ADMIN_IDS`, `HIDDIFY_*`, `USER_PACKAGE_DAYS`, `USER_USAGE_LIMIT_GB`, опционально webhook).
- Новые зависимости и ресурсы: `golang.org/x/image`, ассет(ы) фона карточек, параметр с контактным тегом для поддержки (например `SUPPORT_TAG`).
